13  附录 A：CLAUDE.md 模板

本附录提供两个 CLAUDE.md 模板，分别适用于命令行模式和 Stata MCP 模式。你可以根据自己的使用方式选择对应的模板，复制到项目根目录并根据实际情况修改。

13.1 模板一：命令行模式

适用于没有 Stata MCP 或不想安装 MCP 的用户。

## Stata 数据处理规范

### Stata 配置
- Stata 安装路径：D:/stata18/StataMP-64.exe
  （请替换为你的实际安装路径）
- 运行 dofile 命令格式：
  "D:/stata18/StataMP-64.exe" /e do "script.do"
- 运行前需先 cd 到项目目录，否则日志会写到默认路径

### 路径规范
- 必须使用 Windows 路径格式（D:/...）
- 不能使用 WSL 的 /mnt/d/... 格式
- Stata 无法识别 WSL 路径，会报 r(601) file not found

### 目录约定
- 原始数据放在 data/raw/，不要修改或覆盖
- 清洗后的数据放在 data/processed/
- 代码放在 code/
- 说明文档放在 file/

### 工作流程
- 每次执行 Stata 分三步：Write(.do) → Bash(execute) → Read(.log)
- 运行完成后检查 .log 文件中是否有 r( 开头的错误代码
- 如果有错误，分析原因后修改 .do 文件重新运行

13.2 模板二：Stata MCP 模式

适用于已安装 Stata MCP 的用户（推荐）。

## Stata 数据处理规范

### 调用方式
- 通过 Stata MCP 调用 Stata，不要用命令行
- 使用 stata_run_selection 运行代码片段
- 使用 stata_run_file 运行 .do 文件

### 路径规范
- Stata MCP 运行在 Windows 端，必须使用 Windows 路径格式
  （C:/Users/...），不能用 WSL 的 /mnt/c/... 格式
- 否则会报 r(601) file not found

### 目录约定
- 原始数据放在 data/raw/，不要修改或覆盖
- 清洗过程中以及清洗后的数据存入 data/processed/
- 代码放在 code/
- 说明文档放在 file/

### 输出要求
- 数据处理的代码保存为 .do 文件
- 处理后的数据保存为 .dta 文件
- 数据处理工作小结保存为 .md 文档
  （包含进行了哪些操作，以及最后的描述性统计）

13.3 使用建议

  1. 选择适合你的模板，复制到项目根目录，文件名为 CLAUDE.md
  2. 修改路径信息：将示例路径替换为你本地的实际路径
  3. 补充项目特定需求：比如数据来源说明、特殊变量定义等
  4. 持续完善：在 AI 执行任务后，问它是否遇到了问题，让它总结经验更新 CLAUDE.md
